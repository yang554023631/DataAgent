# 留存率与用户生命周期价值 LTV

留存率和LTV是衡量用户质量和长期价值的核心指标，直接决定投放的ROI和可持续规模。

## 留存率 (Retention Rate)

### 定义
用户在首次使用产品后，经过一段时间后仍然回来使用的比例。

### 常用留存指标

| 留存类型 | 计算公式 | 说明 |
|----------|----------|------|
| **次日留存 (D1)** | 第2天仍活跃的用户数 / 第1天新增用户数 | 衡量新用户首日体验好坏 |
| **7日留存 (D7)** | 第7天仍活跃的用户数 / 第1天新增用户数 | 衡量产品中期留存能力 |
| **30日留存 (D30)** | 第30天仍活跃的用户数 / 第1天新增用户数 | 衡量产品长期粘性 |
| **次月留存** | 下个月仍活跃的用户数 / 当月新增用户数 | 按月统计，适合月度运营分析 |

### 示例计算
1月1日新增用户1000人
- 1月2日活跃350人 → 次日留存 = 350 / 1000 = **35%**
- 1月7日活跃150人 → 7日留存 = 150 / 1000 = **15%**
- 1月30日活跃80人 → 30日留存 = 80 / 1000 = **8%**

### 行业参考值（仅供参考）

| 行业 | 次日留存 | 7日留存 | 30日留存 |
|------|---------|---------|----------|
| 休闲游戏 | 35% - 50% | 15% - 25% | 5% - 12% |
| 重度游戏 | 30% - 45% | 12% - 20% | 5% - 10% |
| 电商APP | 25% - 40% | 10% - 20% | 5% - 10% |
| 教育APP | 20% - 35% | 8% - 15% | 3% - 8% |
| 工具APP | 30% - 50% | 15% - 30% | 8% - 15% |

### 留存率低的常见原因

1. **产品问题**
   - 新手引导太差，用户不知道怎么用
   - 核心功能不好用，用户体验差
   - 性能问题，卡顿、闪退严重

2. **广告问题**
   - 素材过度夸大，与实际产品不符
   - 定向不准，吸引来非目标用户
   - 竞品更有吸引力，用户被抢走

3. **运营问题**
   - 新用户福利不足
   - 没有有效的召回触达
   - 缺乏用户成长体系

### 提升留存的方法

1. **提升次日留存**
   - 优化新手引导，让用户快速体验核心价值
   - 新用户专属福利，鼓励第二天回来领取
   - Push推送召回，提醒用户回来

2. **提升7日留存**
   - 设计7日签到或任务体系
   - 持续更新内容或功能
   - 建立用户成长体系（等级、积分）

3. **提升长期留存**
   - 建立社交关系链
   - 定期运营活动保持活跃度
   - 用户分层运营，针对不同用户做不同触达

## LTV (Life Time Value) 用户生命周期价值

### 定义
一个用户从注册到流失的整个生命周期内，为产品贡献的总价值（通常是总收入）。

### LTV的重要性
LTV决定了**我们最高能接受的获客成本（CAC）**：
- 如果 LTV > CAC → 投放赚钱，可以加大投入
- 如果 LTV < CAC → 投放亏钱，需要优化或停止
- **健康值**：LTV / CAC > 3

### LTV计算方法

#### 方法1：简单计算（历史数据法）
```
LTV = 平均每用户每日收入 × 平均用户生命周期天数
```

**示例**：
- 平均每用户每日收入 = 0.5元
- 平均用户生命周期 = 30天
- LTV = 0.5 × 30 = **15元**

#### 方法2：留存系数法
```
LTV = 首日ARPU × (D1留存 + D2留存 + ... + Dn留存)
```

**示例**：
- 首日ARPU = 2元
- 30天留存率求和 = 35% + 25% + ... + 1% = 3.2
- LTV = 2 × 3.2 = **6.4元**

#### 方法3：付费率法
```
LTV = 平均付费用户价值 × 付费率
```

**示例**：
- 平均付费用户价值 = 100元
- 付费率 = 5%
- LTV = 100 × 5% = **5元**

### 影响LTV的因素

1. **付费率**：有多少用户愿意付费
2. **ARPPU**：平均每个付费用户的付费金额
3. **留存率**：用户能活多久，活得越久贡献越多
4. **LTV回收周期**：多少天能收回获客成本

## LTV与CAC的关系

### 核心公式
```
ROI = LTV / CAC
```

### 不同ROI区间的投放策略

| ROI区间 | 策略 |
|---------|------|
| ROI < 1 | ❌ 停止投放，亏损状态 |
| 1 ≤ ROI < 2 | ⚠️ 谨慎投放，优化降低CAC或提升LTV |
| 2 ≤ ROI < 3 | ✅ 正常投放，保持优化 |
| ROI ≥ 3 | 🚀 加大投放，能跑多少跑多少 |

### LTV回收周期

| 回收周期 | 评价 | 建议 |
|---------|------|------|
| ≤ 7天 | 非常好 | 全力投放 |
| 7-15天 | 好 | 正常投放 |
| 15-30天 | 一般 | 控制投放节奏 |
| > 30天 | 差 | 谨慎投放，注意现金流 |

## 提升LTV的方法

### 1. 提升付费率
- 优化新手付费引导
- 设计首充优惠活动
- 降低付费门槛（如1元首充）

### 2. 提升ARPPU
- 设计更有吸引力的付费内容
- 建立付费成长体系
- 精准推荐，个性化营销

### 3. 提升留存
- 优化产品体验，减少用户流失
- 建立用户召回机制
- 持续更新内容和功能

## 常见误区

### ❌ 误区1：只看首日ROI，不看长期LTV
- 问题：首日ROI可能不赚钱，但7日、30日看是赚钱的
- 后果：过早砍掉了实际上能赚钱的渠道
- 建议：至少看7日LTV，最好看30日

### ❌ 误区2：LTV估算过于乐观
- 问题：用历史最好的数据估算LTV，实际投放达不到
- 后果：投放超预算，最终亏损
- 建议：保守估算，打8折再算

### ❌ 误区3：所有用户用同一个LTV
- 问题：不同渠道、不同定向的用户质量差异很大
- 后果：好渠道没放量，差渠道投太多
- 建议：分渠道、分人群计算LTV，差异化投放

---

## 实战建议

1. **建立LTV预测模型**
   - 用首日或3日数据预测30日LTV
   - 不要等30天后再判断投放效果

2. **分维度计算LTV**
   - 分渠道：不同渠道的用户LTV可能差好几倍
   - 分人群：不同年龄、性别的LTV差异大
   - 分版本：不同APP版本的LTV可能有明显差异

3. **关注LTV趋势**
   - LTV下降比绝对值低更可怕
   - 及时发现LTV下降，排查产品、运营、竞品原因

4. **合理设置KPI**
   - 不要只盯着CPA，CPA低但LTV也低可能更差
   - 最终目标是ROI，不是某个中间指标
