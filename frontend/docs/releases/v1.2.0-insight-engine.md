# v1.2.0 - 洞察规则引擎优化发布

## 发布日期
2026-04-28

## 核心功能

### 1. 双维度洞察分析
- **素材维度（Creative）**：按 creative_id 分组分析
- **广告组维度（Ad Group）**：按 ad_group_id 分组分析
- 两个维度并行执行规则引擎，独立返回洞察结果

### 2. 规则命中率优化
| 规则 | 原阈值 | 新阈值 | 提升 |
|------|--------|--------|------|
| CTR 异常波动 | 2.40%-2.50% | 2.35%-2.55% | ±0.05% |
| CVR 表现优异 | ≥ 3.20% | ≥ 3.00% | -0.20% |
| 最小命中数（min_hits） | 2 | 1 | 适配小广告主 |

**效果**：平均每次查询命中 14 个洞察（原 2-3 个）

### 3. 广告主匹配优化
- **精确匹配**：广告主名称精确匹配
- **相似推荐**：未匹配时返回最相似的 3 个广告主
- **相似度算法**：前缀匹配 > 子串匹配 > 字符集合相似度

### 4. 前端稳定性
- ✅ 白屏修复：所有嵌套属性增加可选链 `?.` 安全访问
- ✅ 重复消息修复：初始化时检查消息长度，避免重复添加

### 5. 用户体验优化
- 洞察按维度分组展示（彩色分组背景 + 维度图标）
- 紧凑卡片模式，信息密度提升
- 默认提示词新增"都有哪些广告主？"快速入口

## 技术架构

### 执行流程
```
nlu_node → advertiser_handle_node → planner_node → executor_node 
    ↓
insight_node（双维度并行） → analyst_node → reporter_node
```

### 双维度洞察实现
```python
# insight_node 内部直接生成两个维度查询
base_query = {
    "index_type": "general",
    "time_range": {...},
    "advertiser_ids": [...],
    "metrics": ["impressions", "clicks", "cost", "conversions"],
}

insight_queries = {
    "creative": {**base_query, "group_by": ["creative_id"]},
    "ad_group": {**base_query, "group_by": ["ad_group_id"]}
}
```
